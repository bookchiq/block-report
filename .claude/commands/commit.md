# Commit Workflow

1. Run `git status` to see what's changed.
2. Run `pnpm build` to make sure everything builds.
3. Stage and commit the changes with a short, descriptive message. **DO NOT MENTION CLAUDE IN THE COMMIT**.
4. Say something nice to the user.
5. Do not push the commit.
